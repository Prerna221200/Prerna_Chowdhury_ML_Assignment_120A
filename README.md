# Cyber Security Incident Detection using Supervised Machine Learning

## 📊 Project Overview
This project focuses on predicting cybersecurity incident response time using supervised machine learning techniques.

## 📁 Dataset
-	Dataset Title: Cybersecurity Incident Dataset
-	Field/Domain: Cybersecurity 
-	Source Link – https://www.kaggle.com/datasets/mustafahabeeb90/cybersecurity-incident-dataset 
- Rows – 100,000
- Columns – 15 Columns
- Features include attack type, duration, severity, data compromised, etc.

## 🎯 Objective
To build regression models that can accurately predict response time and improve incident handling efficiency.

## ❓ Research Questions
1. RQ1 → Classification
2. RQ2 → Feature importance
3. RQ3 → Model comparison
4. RQ4 → Explainability
5. RQ5 → Data imbalance
6. RQ6 → Risk scoring
7. RQ7 → Real-world impact

## ⚙️ Methodology
- Data preprocessing (encoding, scaling)
- Model training (LR, RF, XGBoost, SVM)
- Evaluation using MAE, RMSE, R²
- Feature importance analysis
- Robustness testing


## 📈 Results
- XGBoost achieved best performance
- Feature importance shows attack duration and data compromised are key factors
- Model remains stable under noise and missing data

## 🚀 How to Run
1. Upload dataset to Kaggle
2. Run notebooks RQ1–RQ7
3. Outputs saved in /outputs folder
